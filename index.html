Areas de GIT

Working Directory

Es el area de trabajo donde rara vez vamos a estar utilizando la linea de comandos, salvo por el inicio del proyecto o a lo sumo la eliminacion del mismo. Se refiere a la carpeta donde iniciamos el repositorio y basicamente donde vamos a crear, modificar y eliminar los archivos que formen parte de nuestro proyecto

  • dir || ls Muestra el contenido de la carpeta actual
  • mkdir <name Crea una carpeta con el nombre indicado
  • cd <pathAbre el directorio especificado
  • git init Inicializa un repositorio de git
  • rm -rf.gitElimina el repositorio de manera forzada

Staging Area (INDEX)

El area de control de cambios, corresponde a un archivo que se encuentra en la carpeta .git, y se utiliza para realizar las capturas de pantalla del codigo, tambien conocidas como "snapshot" de los diferentes archivos del proyecto. Aqui se realiza el seguimiento de los mismos, para realizar la confirmacion y poder acceder a los diferentes cambios dentro de la historia de nuestro proyecto. Es importante tener en cuenta que todo archivo que no sea agregado al INDEX, no se podra recuperar en caso de ser eliminado.

  • Git statusMuestra el estado de los archivos con respecto a la ultima confirmacion, en caso que exista.
    • untrackedAl archivo no se le esta realizando seguimiento(no fue agregado al index)
    • stagedLos cambios fueron agregados pero aun no estan confirmados ( no se hixo el commit)
    • modified: El archivo difiere con respecto a lo almacenado en la ultima confirmacion
    • commited Los cambios fueron almacenados en una confirmacion
  • git add <fileAgrega al INDEX el/los archivos seleccionados
  • git rm --cached <file>Quita del INDEX el archivo seleccionado
  • git restore --staged <file>remueve los cambios de los archivos agregados al INDEX

Repository

Area donde se almacenan los cambios, mediante confirmaciones, en formato BLOB (Binary Object File). Solamente se confirman las modificaciones agregadas al INDEX; y mediante apuntadores podemos navegar dentro del historial.

  • git commitabre el editor para realizar la confirmacion
  • git blame <file muestra linea por linea los autores de las confirmaciones, en caso que existan
  • git log Muestra el historial de confirmaciones en orden descendente
  • git push <remote> <url>envia a un repositorio remoto los cambios
  • git remote add <alias> <url>agrega una direccion remota asignandole un alias
